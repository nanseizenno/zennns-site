# 多个 PCN 如何形成状态迁移前置控制网络？

> 基础概念可参照：[TPCA](/zh/concepts/tpca/)、[PCN](/zh/concepts/pcn/)、[CAE-SDB](/zh/concepts/cae-sdb/)、[C/A/E](/zh/concepts/cae/)、[S/D/B](/zh/concepts/sdb/)。

单个 [PCN](/zh/concepts/pcn/) 对应一个目标状态入口。

多个 PCN 按状态迁移关系连接起来，就形成状态迁移前置控制网络。

在产线布局、自动化单元设计、物流路径规划、MES / WCS 联动设计时，如何判断哪里需要设置 PCN 点。

PCN 网络的核心，不是把多个设备画成一张连接图，而是把复杂制造系统中的状态迁移入口显式标出来。

当工程师讨论产线方案时，如果某个位置存在明确的“当前状态 → 目标状态”，并且进入目标状态前需要综合条件、许可和执行链进行判断，就可以直接提出：

**这里需要增加一个 PCN 点。**

这句话的含义是：这里不只是需要一个传感器、一个 interlock 或一个报警，而是需要一个目标状态进入前的前置判定节点。

---

## 1. PCN 网络按状态入口组织

PCN 网络不是按设备名称组织，而是按目标状态入口组织。

例如：

| 状态入口 | PCN 点 |
|---|---|
| 机器人等待 → 抓取 | 机器人抓取前 PCN |
| 放置完成 → 压装 | 压装前 PCN |
| 待检测 → 检测执行 | 检测前 PCN |
| 任务存在 → 任务派发 | 任务派发前 PCN |
| AGV 到站 → 站点承接 | 站点承接前 PCN |
| 包装待机 → 包装执行 | 包装前 PCN |
| 包装完成 → 码垛 | 码垛前 PCN |
| 码垛完成 → 入库 / 交接 | 入库 / 交接前 PCN |

设备是载体，状态入口才是节点。

因此，PCN 网络不是“设备网络”，而是“状态迁移入口网络”。

---

## 2. 判断是否需要增加 PCN 点

某个位置是否需要设置 PCN 点，可以用以下问题判断：

1. 这里是否存在明确的当前状态和目标状态？
2. 进入目标状态前，是否需要多个状态信号共同判断？
3. 是否存在安全许可、上位许可、资源锁、区域许可或人工确认？
4. 进入目标状态后，下游是否必须承接？
5. 是否存在回流、异常分流、降级、等待或禁止进入等多路径控制？
6. 这个判断结果是否需要记录、复盘或后续改善？

如果多数回答为“是”，这里就不是普通信号点，而是 PCN 点候选位置。

例如，包装前不只是判断包装机 ready，还要判断待包装品、包装材料、质量放行、下游码垛承接和结果记录是否成立。

码垛前不只是判断码垛设备 ready，还要判断包装完成状态、托盘状态、码垛规则、下游入库或交接路径是否成立。

入库 / 交接前不只是判断搬送设备 ready，还要判断仓储位置、交接许可、标签信息、结果回写和异常退回路径是否成立。

这些位置都适合用 PCN 来表达。

## 3. PCN 之间的三类连接

PCN 网络中的连接，不只是通信连接，更重要的是状态迁移关系。

### 状态连接

一个 PCN 的输出状态，成为另一个 PCN 的输入状态。

例如：

抓取完成 → 投放前 PCN；  
包装完成 → 码垛前 PCN；  
码垛完成 → 入库 / 交接前 PCN。

### 许可连接

一个 PCN 的判定结果，影响另一个 PCN 的许可状态。

例如：

资源锁未释放，阻止任务派发；  
站点不可接收，阻止 AGV 进入；  
包装确认未完成，阻止码垛；  
入库交接许可未成立，阻止物料进入仓储路径。

[A 许可](/zh/concepts/cae/) 在 PCN 网络中是关键约束，不应被当作普通条件项处理。

### 执行链连接

一个 PCN 进入目标状态前，需要确认后续执行链是否接得住。

例如：

抓取前要看投放位、回流路径和异常分流；  
包装前要看码垛承接；  
码垛前要看入库 / 交接路径；  
检测前要看结果回写和质量判定链路。

如果只看单体 ready，[执行链](/zh/concepts/cae/) 就会断开。

## 4. PCN 网络的工程价值

PCN 网络的价值，不是增加一层复杂图纸，而是让工程师能用统一语言讨论状态迁移问题。

以前讨论产线布局时，常见说法是：

这里加一个传感器；  
这里加一个互锁；  
这里加一个报警；  
这里让 HMI 显示一下；  
这里等 MES 放行。

这些说法往往停留在局部信号或局部动作。

PCN 语言进一步追问：

这里是不是一个目标状态入口？  
进入前需要哪些条件？  
哪些许可必须成立？  
执行链是否接得住？  
不能进入时走哪条路径？  
这个判断是否需要记录？

因此，“这里需要增加 PCN 点”可以成为产线布局、自动化联动和系统协同设计中的工程讨论用语。

这句话把讨论对象从“加一个信号”提升为“定义一个状态迁移前置判定节点”。

---

## 小结

PCN 是目标状态入口前的前置判定节点。

多个 PCN 按状态迁移关系连接，就形成 PCN 网络。

PCN 网络不是设备连接图，而是状态迁移入口图。

它帮助工程师识别：

哪里需要前置判定；  
哪里存在许可边界；  
哪里执行链可能断开；  
哪里需要记录和复盘；  
哪里应当增加新的 PCN 点。

当工程师能够在产线布局讨论中自然说出：

**这里需要增加 PCN 点。**

PCN 就不再只是理论概念，而成为复杂制造系统状态迁移设计的一种共同工程语言。
