TPCA / CAE-SDB 讨论的是一个很具体的工程问题：

系统准备进入目标状态、目标执行路径或目标物理执行阶段之前，如何判断它是否具备进入条件；如果不能进入，问题来自哪里，下一步应走哪条控制路径。

TPCA 是架构，CAE-SDB 是判定逻辑，PCN 是这套架构落到工程现场的最小单元。

没有 PCN，TPCA 仍然可以作为一种方法论被说明；但工程现场并不会因为方法论存在，就自动知道应该在哪里取信号、在哪里判定、在哪里输出控制路径、在哪里记录状态。

有了 PCN，问题才会落到具体位置：

这个判定节点设在哪里；  
它面对哪个当前状态；  
它准备判断哪个目标状态；  
它读取哪些多源状态信号；  
哪些信号映射到 C 条件、A 许可、E 执行链；  
这些状态从 S 结构、D 动态、B 边界上如何判定；  
判定之后输出哪条控制路径；  
本次判定如何进入记录和复盘。

这就是 PCN 成为 TPCA 最小工程单元的原因。

---

## 1. TPCA 是架构，PCN 是状态入口上的工程节点

TPCA，Transition Pre-Control Architecture，状态迁移前置控制架构，定义的是一类工程处理方式。

它关注系统进入目标状态之前的前置判定过程。基本链条可以概括为：

当前状态  
→ 目标状态  
→ 多源状态信号  
→ C/A/E 状态映射  
→ S/D/B 判定  
→ CAE-SDB 判定结果  
→ 多路径控制输出  
→ 状态记录与追溯

这条链条说明了系统应如何处理“不能进入下一状态”的问题。

但架构本身不能直接运行。现场还需要知道：

哪个位置需要前置判定；  
哪些信号需要纳入判断；  
哪个 ready 只代表本机状态；  
哪个许可必须单独确认；  
哪个下游状态属于执行链承接；  
哪个结果应进入 HMI、MES、WCS 或日志。

因此，TPCA 必须通过一个工程节点落地。这个节点就是 PCN。

PCN，Pre-Control Node，前置控制节点，部署在目标状态进入前。它接收多源状态信号，完成 C/A/E 状态映射与 S/D/B 判定，并输出多路径控制结果和状态记录。

CAE-SDB 是 PCN 内部使用的判定逻辑。

C/A/E 说明问题来自哪个状态变量域：

C = Condition，条件状态；  
A = Authority，许可状态；  
E = Execution Chain，执行链状态。

S/D/B 说明该状态变量域的问题属于什么性质：

S = Structure，结构完整性；  
D = Dynamics，动态时序有效性；  
B = Boundary，控制边界。

例如，视觉结果可以属于 C 条件状态，安全许可可以属于 A 许可状态，下游可承接状态可以属于 E 执行链状态。进一步看，视觉结果未接入属于 C-S，视觉结果过期属于 C-D，安全许可未成立属于 A-B，下游承接状态未刷新属于 E-D。

CAE-SDB 可以把模糊 NG 展开成结构化判定结果。PCN 则进一步明确：这个判定逻辑在哪个状态入口触发，读取哪些状态，如何转成控制路径，以及如何进入记录。

## 2. PCN 的最小闭环：位置、输入、判定、输出、记录

一个 PCN 至少需要明确以下内容。

| 项目 | 说明 |
|---|---|
| 当前状态 / 当前阶段 | 系统现在处于什么状态 |
| 目标状态 / 目标阶段 | 系统准备进入什么状态 |
| PCN 设置位置 | 判定节点设置在目标状态入口前的哪个位置 |
| 多源状态信号 | 来自设备、PLC、机器人、视觉、安全、MES、WCS、HMI 或其他系统的状态 |
| C/A/E 映射 | 哪些状态属于条件，哪些属于许可，哪些属于执行链 |
| S/D/B 判定 | 状态是否存在结构缺失、动态异常或控制边界问题 |
| 多路径控制输出 | 允许进入、等待、重识别、回流、异常分流、下游协调、禁止进入、安全锁定、人工确认等 |
| 状态记录 | 记录输入、判定结果、控制路径和追溯信息 |

这个表就是 PCN 的最小分析模板。

PCN 之所以是最小工程单元，不是因为它功能最少，而是因为它包含了 TPCA 落地所需的最小闭环。

这个闭环包括五件事：

第一，PCN 有明确的目标状态入口。  
没有目标状态，就没有“进入前”的判定对象。机器人等待阶段进入抓取阶段、放置完成阶段进入压装阶段、任务池进入任务派发路径、车辆在线进入可持续协同搬送状态，都可以设置 PCN。

第二，PCN 有多源状态输入。  
现场的“不能进入”通常不会只来自一个信号。机器人 ready、视觉 OK、安全许可、下游承接、上位放行、资源锁、工件状态、路径状态、结果回写，可能分布在不同设备和系统中。

第三，PCN 有 C/A/E 状态映射。  
多源状态信号不能只作为散乱信号列表存在。PCN 需要判断它们在目标状态进入前承担什么作用：条件、许可，还是执行链。

第四，PCN 有 S/D/B 判定。  
同一个条件状态，可能存在结构缺失、动态不可信或控制边界问题。同一个许可状态，可能是来源未定义、许可撤销或触发禁止进入边界。同一个执行链状态，可能是下游路径未定义、反馈未刷新或需要协调、回流、隔离。

第五，PCN 有控制输出和状态记录。  
判定结果需要转成控制路径。判定过程需要形成记录。没有输出，PCN 只停留在诊断说明；没有记录，PCN 难以用于复盘、改善和复用。

## 3. PCN 与 interlock、报警、HMI / MES / WCS 的关系

在很多自动化系统中，interlock 已经存在。interlock 可以阻止错误动作，通常回答“允许 / 不允许”“条件成立 / 条件未成立”。

PCN 的处理范围更大。

它关注目标状态进入前的结构化判定过程，进一步回答：

不允许进入来自哪个变量域；  
问题属于结构、动态还是边界；  
是否涉及关键许可；  
执行链是否能接续；  
下一步应等待、重识别、回流、协调、降级、禁止、安全锁定还是人工确认；  
本次判定如何记录；  
同类问题下次如何复用。

PCN 可以读取 interlock 的结果，也可以把 interlock 作为 A 许可状态或 C 条件状态的一部分。

报警通常说明某个异常已经发生。PCN 关注的是进入目标状态之前的判定。以机器人抓取为例，抓取失败后的报警是动作后的结果；PCN 关心的是进入抓取阶段之前，工件、视觉、安全、机器人路径、夹爪、下游承接、异常分流、结果回写等状态是否已经具备。

PCN 也可以部署在不同系统层级中。

在 PLC / HMI 场景中，PCN 可以对应自动化执行单元进入目标物理阶段前的判定节点。  
在 MES / WCS 场景中，PCN 可以对应任务派发、资源放行、站点承接、群体协同搬送恢复等目标执行路径前的判定节点。  
在生产 DX 场景中，PCN 可以对应人员、设备、系统、流程之间的状态迁移入口。

HMI、MES、WCS 可以显示 PCN 的判定结果。PLC、边缘控制器、WCS 或上位系统可以承接 PCN 的控制输出。日志系统和数据平台可以记录 PCN 的状态履历。

PCN 是一个工程节点概念，具体形态可以随系统层级变化，但基本结构保持一致：

目标状态入口；  
多源状态输入；  
C/A/E 映射；  
S/D/B 判定；  
多路径控制输出；  
状态记录。

## 4. 一个例子：机器人进入抓取阶段前的 PCN

以视觉识别输送线机器人分拣单元为例。

当前状态是机器人处于等待阶段，工件进入视觉识别区域，系统准备判断是否进入抓取阶段。

目标状态是机器人进入抓取阶段。

PCN 设置在机器人抓取动作启动之前。

多源状态信号可以包括：视觉识别结果、工件存在状态、工件位置和姿态、识别结果有效性、机器人自动模式、机器人 ready、机器人路径可达状态、安全许可、主输送带状态、返回输送带状态、异常输送带状态、正常投放位可接收状态、上位系统放行状态。

这些信号可以映射为：

C 条件：工件存在、视觉结果有效、位置和姿态满足抓取要求；  
A 许可：安全许可、机器人自动模式、上位系统放行、区域许可；  
E 执行链：机器人路径可达、夹爪可用、正常投放位可承接、返回路径可用、异常分流路径可用、结果回写可用。

S/D/B 判定进一步判断这些状态是否定义并接入，是否有效、同步、可信，当前是否需要等待、重识别、回流、异常分流、禁止进入或人工确认。

控制输出可能包括：允许抓取、等待、重识别、重定位、放弃本次抓取并回流、异常品分流、禁止进入抓取阶段、安全锁定、人工确认、状态记录。

这个例子说明：PCN 不在机器人失败后解释报警，而是在机器人进入抓取阶段前，完成目标阶段入口的结构化判定。

## 5. PCN 履历与 PCN 网络

PCN 不只输出当前控制路径，还形成状态履历。

PCN 履历记录当前状态、目标状态、输入信号、C/A/E 映射结果、S/D/B 判定结果、控制路径、时间信息和追溯编号。

普通日志通常记录“发生了什么”。

PCN 履历进一步记录：

系统为什么不能进入目标状态；  
哪个变量域不成立；  
问题属于什么性质；  
当时选择了哪条控制路径；  
同类问题是否反复出现。

这类履历可以用于复盘和改善。例如，某个目标状态入口经常失败，某类许可经常未成立，某类下游承接经常不同步，某个执行链边界经常触发异常分流，某些经验判断需要标准化，某些状态信号需要补充定义。

单个 PCN 对应一个目标状态入口。多个 PCN 连接起来，就形成状态迁移前置控制网络。

在一条自动化产线中，PCN 可以分布在工件进入识别前、机器人进入抓取前、机器人进入投放前、压装单元进入压装前、检测单元进入检测前、异常品进入分流前、AGV 进入站点前、缓存区释放前、MES / WCS 派发任务前、下游工位接收前。

多个 PCN 的履历积累后，可以显示：哪一类状态迁移入口最容易失败，条件问题集中在哪些节点，许可问题集中在哪些区域，执行链问题集中在哪些下游承接环节，哪些问题属于结构缺失，哪些问题属于动态不可信，哪些问题属于边界处理不清。

这就是 PCN 网络的价值。

它让复杂制造系统中的状态迁移问题，从零散经验转化为可观察、可比较、可复盘的工程结构。

---

## 小结

TPCA 是状态迁移前置控制架构。  
CAE-SDB 是 PCN 内部的判定逻辑。  
PCN 是 TPCA 的最小工程单元。

PCN 把方法论落到了现场可识别的位置：

在哪里判断；  
判断什么目标状态；  
读取哪些状态；  
如何映射 C/A/E；  
如何判定 S/D/B；  
输出哪条控制路径；  
如何记录和复盘。

没有 PCN，TPCA 主要停留在方法论说明层。

有了 PCN，TPCA 才能进入工程现场，变成一个个目标状态入口前的前置判定节点。

从单个 PCN 到多个 PCN 网络，TPCA / CAE-SDB 才能逐步形成复杂制造系统中的状态迁移前置控制结构。
