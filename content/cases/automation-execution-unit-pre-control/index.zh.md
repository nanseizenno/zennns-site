## 为什么 Robot Ready，不代表可以进入抓取阶段

## 1. 现场问题

在视觉识别输送线机器人单元中，现场经常会遇到一种情况：

视觉系统已经输出 OK。  
机器人显示 Ready。  
安全回路没有报警。  
PLC 侧部分条件看起来也成立。  

但机器人仍然不能稳定进入抓取阶段。

有时系统继续等待。  
有时工件错过抓取窗口，只能进入回流。  
有时机器人进入动作后发现下游不可承接。  
有时异常品路径不可用。  
有时现场只能看到“抓取 NG”“等待中”“条件未满足”，但无法快速判断问题到底在哪里。

这个问题的关键不在某一个设备是否 Ready。

机器人准备进入的，不是一个单体动作，而是一条物理执行路径。

这条路径至少包括：

- 识别结果是否仍然有效；
- 工件是否处于可抓取位置；
- 安全和区域许可是否成立；
- 机器人路径是否可达；
- 夹爪是否可用；
- 正常投放位是否可承接；
- 异常品是否有分流路径；
- 未抓取工件是否可以回流；
- 必要结果是否可以记录或回写。

任何一段不成立，系统都不应简单地把问题压缩成“机器人不动作”或“抓取 NG”。

TPCA / CAE-SDB 的作用，是在机器人进入抓取阶段之前，把这个入口拆开判断。

PLC / PCN 先确认：

C：抓取条件是否成立；  
A：系统是否允许进入抓取阶段；  
E：抓取后的执行链是否能够接续。  

然后进一步判断：

S：所需信号、接口和路径是否已经定义并接入；  
D：当前信号是否有效、同步、未超时；  
B：边界状态下应进入哪条控制路径。  

最终输出不只是一句“允许 / 不允许”，而是正常抓取、回流再识别、异常分流、等待、重定位、人工确认、禁止进入或安全锁定等控制路径。

## 2. 案例对象与 PCN 位置

本案例以视觉识别输送线机器人单元作为代表例。

工件由主输送带连续流入。视觉系统对工件进行识别，输出工件有无、位置、姿态、置信度、异常判定和结果时间戳。PLC / PCN 在机器人进入抓取阶段之前，读取视觉系统、机器人控制器、安全装置、输送带、正常投放位、返回输送带、异常输送带和上位系统的状态信号。

根据判定结果，工件可能进入以下路径：

- 正常抓取并投放到规定位置；
- 进入返回输送带，重新进入识别循环；
- 进入异常输送带；
- 等待下一次判定；
- 触发人工确认；
- 禁止进入抓取阶段；
- 执行安全锁定。

![视觉识别输送线机器人单元前置控制示意图](/images/tpca/06-pcn-pick-flow.png)

图：PLC / PCN 在机器人进入抓取阶段前，综合视觉系统、机器人控制器、安全装置、输送带、正常投放位、返回路径、异常分流路径和上位系统等多源状态信号，进行 C/A/E 映射与 S/D/B 判定，并输出正常抓取、回流、异常分流、等待、禁止进入或安全锁定等控制路径。

当前物理执行阶段：

机器人处于等待阶段，主输送带持续输送工件，视觉系统完成识别，PLC / PCN 准备判断是否允许机器人进入抓取阶段。

目标物理执行阶段：

机器人进入抓取阶段，并使工件进入正常投放、返回循环、异常分流、等待确认或禁止进入等路径之一。

PCN 的位置在目标阶段入口前。

![PCN 在目标阶段入口前的位置关系图](/images/tpca/07-pcn-position-before-target-stage.png)

图：PCN 部署在当前阶段与目标阶段之间，不放在动作失败之后，而放在进入目标物理执行阶段之前，用于决定系统进入抓取、投放、回流、异常分流、等待、禁止进入或安全锁定等路径。

这个位置很关键。

PCN 放在动作失败之后，只能做事后诊断。  
PCN 放在目标阶段入口前，才能决定系统是否进入、等待、回流、分流、禁止或锁定。

本文关注的是自动化执行单元进入目标物理执行阶段前的前置判定。视觉识别、机器人轨迹、夹爪动作和输送带控制都可以作为现场实现的一部分，但不是本文重点。

## 3. 多源状态信号与 C/A/E 映射

PLC / PCN 不只读取一个 ready 信号，而是读取与目标阶段进入相关的多源状态信号。

| 信号来源 | 代表性信号 |
|---|---|
| 视觉系统 | 工件存在、位置坐标、姿态角度、识别置信度、异常判定结果、识别结果时间戳、结果有效标志、工件跟踪 ID |
| 主输送带 | 运行状态、速度、编码器位置、当前位置推定、工件是否进入抓取窗口 |
| 返回输送带 | 运行状态、可接收状态、回流路径是否可用 |
| 异常输送带 | 运行状态、可接收状态、异常品排出路径是否可用 |
| 正常投放位 | 是否空位、是否可承接、是否完成前一件处理 |
| 机器人控制器 | 自动模式、ready、当前位置、路径可达、程序状态、动作完成、夹爪 / 真空状态、报警状态 |
| 安全装置 | 安全门、光栅、急停、安全回路、区域进入许可 |
| 上位系统 / HMI | 工单信息、生产许可、异常品处理许可、人工确认、结果记录或回写要求 |

这些信号进入 PCN 后，不直接生成控制结论。PCN 先判断这些信号在目标阶段入口前分别承担什么作用，再进入 C/A/E 映射。

| 变量域 | 判定问题 | 本案例中的代表项 |
|---|---|---|
| C：Condition 条件状态 | 工件是否构成可抓取对象 | 工件存在、位置合格、姿态合格、视觉结果有效、置信度满足、工件跟踪 ID 与输送带位置一致 |
| A：Authority 许可状态 | 系统是否允许进入抓取阶段 | 安全许可、区域许可、机器人自动模式、PLC 放行许可、上位许可、异常品处理许可、人工确认 |
| E：Execution Chain 执行链状态 | 抓取后执行链是否能够接续 | 机器人路径可达、夹爪可用、正常投放位可承接、异常输送带可接收、返回输送带可接收、结果记录或回写链路可用 |

这里有两个关键点。

第一，A 许可可以构成独立必要约束。

安全门未关闭、光栅触发、急停触发、区域许可未成立或上位系统禁止放行时，即使工件条件成立、机器人 ready、下游看起来可用，也不得允许机器人进入抓取阶段。

第二，E 执行链不等于 Robot Ready。

Robot Ready 只是机器人本体状态。E 还包括抓取后的投放、异常分流、回流、下游承接和结果记录。

```text
Robot Ready ≠ 抓取执行链成立
```

机器人 ready，但异常输送带不可接收，异常品路径仍不成立。  
机器人 ready，但正常投放位未清空，正常投放路径仍不成立。  
机器人 ready，但返回输送带不可用，错过抓取窗口后的回流路径仍不成立。  
机器人 ready，但视觉结果已经过期，C 条件仍不成立。

这就是很多现场“Ready 了仍不动作”的结构原因。

## 4. S/D/B 性质判定

S/D/B 用于判断 C/A/E 中的问题属于结构、动态还是边界。

| 判定性质 | 判定问题 | 本案例中的代表项 |
|---|---|---|
| S：Structure 结构完整性 | 需要的信号、接口、映射关系和路径是否已经定义并接入 | 视觉结果是否接入 PLC；视觉坐标与机器人坐标是否建立映射；返回输送带和异常输送带是否纳入控制逻辑；安全许可是否接入 A 域 |
| D：Dynamics 动态时序有效性 | 当前信号是否有效、同步、未超时、未冲突 | 视觉结果是否过期；工件位置是否与输送带追踪同步；安全许可是否瞬时撤销；机器人 ready 是否延迟；下游状态是否未刷新 |
| B：Boundary 控制边界 | 边界状态下应进入哪条路径 | 位置偏差接近补偿边界；识别置信度处于边界；抓取窗口即将错过；异常路径暂时不可接收；是否回流、等待、重识别或人工确认 |

S/D/B 的价值在于，它不把问题简单写成“NG”。

同样是不能抓取，性质可能完全不同。

- 视觉结果没有接入 PLC：C-S，结构问题；
- 视觉结果已超时：C-D，动态问题；
- 视觉置信度接近边界：C-B，控制边界问题；
- 安全门打开：A-B，许可边界阻断；
- 下游状态未刷新：E-D，执行链动态问题；
- 异常输送带未接入：E-S，执行链结构问题；
- 异常输送带短时不可接收：E-B，执行链边界问题。

同一个“抓取 NG”，进入 PCN 后会变成不同的工程处置路径。

## 5. 多路径控制输出

PCN 的输出应直接服务于现场控制路径。

| 现场状态 | 前置判定结果 | 控制路径 |
|---|---|---|
| 工件存在，视觉结果有效，安全许可成立，机器人路径可达，下游可承接 | C/A/E 成立 | 允许进入抓取阶段，正常抓取并投放 |
| 工件位置接近补偿边界 | C-B | 重定位、低速接近、二次确认或等待下一次识别 |
| 视觉结果已过期 | C-D | 重识别、重采样或回流再识别 |
| 识别置信度不足，但工件仍可回流 | C-B + E-B | 放弃本次抓取，使工件进入返回输送带 |
| 安全门打开、光栅触发或急停触发 | A-B | 禁止进入抓取阶段或安全锁定 |
| 上位系统未放行 | A-B | 等待许可、禁止进入或人工确认 |
| 正常投放位不可承接 | E-B | 等待、下游协调、回流或暂缓进入抓取阶段 |
| 异常输送带不可接收 | E-B | 等待、回流、暂停异常分流或人工确认 |
| 返回输送带状态未接入 | E-S | 禁止进入回流路径，提示执行链结构缺失 |
| 多个关键条件同时异常 | C/A/E 多域异常 | 禁止进入、异常记录或人工确认 |

这些输出说明，PCN 不是把所有失败都显示成“抓取 NG”。

它把不同问题分流到不同路径：

```text
允许进入
等待
重识别
重采样
重定位
回流
异常分流
下游协调
禁止进入
安全锁定
人工确认
增强记录
```

现场人员看到的不再只是一个结果，而是一个可执行的下一步。

## 6. 典型判定与 HMI 显示

以“视觉 OK、Robot Ready，但机器人没有抓取”为例。

传统画面可能只显示：

```text
视觉 OK
Robot Ready
等待中
抓取条件未满足
```

现场仍需要人工查：

视觉结果是否过期；  
工件是否已经错过抓取窗口；  
安全许可是否瞬时撤销；  
正常投放位是否可承接；  
异常输送带是否可用；  
返回输送带是否可接收；  
上位系统是否放行；  
机器人路径是否仍可达。  

PCN 的判定过程可以整理为：

```text
当前阶段：等待 / 识别完成
目标阶段：进入抓取阶段

C 条件：
视觉结果存在，但结果时间戳已过有效窗口 → C-D

A 许可：
安全许可成立，上位许可成立 → A 满足

E 执行链：
返回输送带可接收，正常抓取窗口已错过 → E-B

判定结果：
当前不允许进入抓取阶段

控制输出：
放弃本次抓取，使工件进入返回输送带，并触发下一轮识别
```

这里的重点不是“机器人为什么不动作”，而是系统已经选择了更合适的路径：

```text
不抓取 → 回流 → 再识别
```

HMI 上可以不只显示“抓取 NG”，而显示结构化结果。

示例一：

```text
当前阶段：识别完成 / 等待抓取
目标阶段：机器人抓取阶段

判定结果：不允许进入抓取阶段

主要原因：
C-D：视觉结果已超出有效时间
E-B：工件仍可进入返回路径

控制输出：
回流再识别

TraceLogID：
PCN-PICK-2026-0001
```

示例二：

```text
当前阶段：等待抓取
目标阶段：机器人抓取阶段

判定结果：禁止进入

主要原因：
A-B：安全区域许可未成立

控制输出：
禁止进入 / 安全锁定

TraceLogID：
PCN-PICK-2026-0002
```

这种显示方式能把现场排查从“查所有信号”变成“按 C/A/E 和 S/D/B 查看当前阻断点”。

## 7. 状态记录、复盘与案例意义

PCN 在输出控制路径时，应同步记录判定过程。

代表性记录内容包括：

- 当前物理执行阶段；
- 目标物理执行阶段；
- 输入信号及来源；
- 信号时间戳；
- C/A/E 映射结果；
- S/D/B 判定结果；
- 控制路径；
- 人工确认结果；
- TraceLogID；
- 后续处理结果。

这些记录可以用于：

- 现场复盘；
- HMI 诊断显示；
- 高频问题统计；
- 新人培训；
- 类似单元模板复用；
- 后续改善前后对比。

记录的重点不是保存所有原始数据，而是保存“为什么没有进入抓取阶段”或“为什么进入了回流 / 异常分流 / 安全锁定路径”的判定结构。

本案例说明，自动化执行单元进入目标物理执行阶段前，不能只看单体 ready、普通 interlock 或报警状态。

在视觉识别输送线机器人单元中，Robot Ready 只说明机器人本体可运行。系统能否进入抓取阶段，还需要判断工件条件、安全和上位许可、抓取后的投放路径、异常分流路径、返回路径和结果记录链路是否成立。

TPCA / CAE-SDB 的作用，是把“抓取 NG”“等待中”“条件未满足”这类压缩状态，展开为：

```text
C：条件是否成立
A：许可是否成立
E：执行链是否可接续

S：结构是否完整
D：动态是否可信
B：边界是否可控
```

然后输出对应控制路径。

这样，自动化执行单元的状态迁移判断就从“信号堆叠”和“人工查梯形图”，转化为目标阶段入口前的结构化判定。

其直接价值在于：

- 让 Robot Ready 的边界变清楚；
- 让“不能进入下一阶段”的原因可拆解；
- 让回流、重识别、异常分流、等待、禁止进入、安全锁定等路径显式化；
- 让 HMI 显示和状态履历具备统一结构；
- 让类似自动化单元的判定逻辑更容易复用。

---

## 8. 公开边界

本文为公开说明案例，用于解释 TPCA / CAE-SDB 在自动化执行单元中的工程逻辑。

实际项目中的具体设计，应根据设备结构、现场节拍、安全要求、控制平台和交付规范另行确定。
