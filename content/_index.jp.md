## 複雑な工程システムにおける状態遷移方法論

複雑な工程システムが目標状態へ移行する前には、一連の状態判定が必要となる。

これらの判定は、製造システム、自動化実行ユニット、PLC / HMI、ロボットシステム、MES / WCS、群制御システム、デジタル呼び出しシステムなど、幅広い対象に存在する。

工程現場では、次のような問題が長年にわたり続いている。

- 設備は Ready 状態にあるが、システムが次の段階へ進めない。
- PLC の条件は成立しているが、実行機構が動作しない。
- MES / WCS には Waiting、Pending、Blocked と表示されるが、停滞理由までは説明できない。
- 同一の現象に対し、担当者によって判断が異なる。
- アラームは設備の位置情報は示せても、状態遷移が失敗した理由までは説明できない。
- プロジェクトは稼働しているが、状態遷移条件・異常境界・実行チェーンの設計が個人の経験に依存している。
- 新規プロジェクト、新規設備、新規チームの間で、状態遷移設計に関する共通基準が不足している。

これらの問題は、多くの場合、同一の局面で発生する。  
すなわち、システムが目標状態、目標実行経路、あるいは目標物理実行段階へ遷移する直前である。

## 状態遷移を独立した設計対象として扱う理由

工程制御では、一般に次のような状態表現が用いられる。

- OK / NG
- Ready / Not Ready
- True / False
- Waiting
- Pending
- Blocked
- Interlock
- Alarm

これらの状態表現は、制御ロジックを成立させるうえで必要不可欠である。  
しかし、現場が本質的に知りたい問いに対しては、必ずしも十分な答えを与えない。

- なぜ目標状態へ遷移できないのか。
- 問題は条件・許可・実行チェーンのいずれに起因するのか。
- 現在の問題は、構造欠落・動的異常・制御境界のいずれに分類されるのか。
- 次に選択すべきは、待機・再試行・回流・協調・縮退・人手による確認のいずれか。
- 判断過程をどのように記録・追跡し、継続的改善へつなげるのか。

これらの課題は、制御ロジック自体の不足に起因するとは限らない。  
むしろ、状態遷移条件、異常境界、実行チェーンの受け渡し、判断記録が、プロジェクトごとに担当者個人の経験に委ねられて整理されてきたことが大きな要因である。

担当者・プロジェクト・設備が変われば、設計粒度や判断基準も変化しやすくなる。  
プログラムが正常に稼働していても、状態遷移条件の十分性、異常境界の明確さ、実行チェーンの引き継ぎ可否は、依然として個人の経験に依存しがちである。

ここに、状態遷移を独立した設計対象として扱う必要性がある。

## TPCA / CAE-SDB

TPCA（Transition Pre-Control Architecture）は、状態遷移前の工程判定を扱うための工学的方法論である。

CAE-SDB は、TPCA の中核を成す判定構造である。

基本処理は、以下の流れで構成される。

現在状態 / 現在段階  
↓  
目標状態 / 目標段階  
↓  
多源状態信号  
↓  
C / A / E 状態マッピング  
↓  
S / D / B 判定  
↓  
複数経路制御出力  
↓  
状態記録と追跡

C / A / E は、以下の三つの状態変数を表す。

- Condition（条件状態）
- Authority（許可状態）
- Execution Chain（実行チェーン状態）

S / D / B は、以下の三つの判定性質を表す。

- Structure（構造的完全性）
- Dynamics（動的・時系列有効性）
- Boundary（制御境界）

TPCA / CAE-SDB は、Ready、NG、Waiting といった圧縮された状態表現をさらに展開し、問題の発生領域、判定性質、制御経路、記録方式を明確化する。

## 読み進め方

### Engineering Questions｜工程上の問い

次のような問いを探している場合は、まず Engineering Questions から読み始めることを推奨する。

- なぜ Ready 状態だけでは不十分なのか。
- なぜ Waiting の原因調査は困難になるのか。
- なぜ PLC が Ready であるにもかかわらず動作しないのか。
- なぜ MES は記録は可能でも、停滞理由までは説明できないのか。
- なぜアラームが増加しても、調査時間は十分に短縮されないのか。

### Concepts｜概念集

状態遷移に関する工程言語を体系的に整理したい場合は、概念集を参照されたい。

ここでは、TPCA、PCN、CAE-SDB、Condition、Authority、Execution Chain、Structure、Dynamics、Boundary、Control Path といった中核概念を整理する。

### Cases｜ケース

工程上の適用事例を確認したい場合は、ケースを参照されたい。

ケースでは、PLC / HMI、ロボットユニット、MES / WCS、AGV / AMR 群制御システム、AI Gateway、デジタル呼び出しガバナンスなどの応用領域を扱う。

### White Papers｜ホワイトペーパー

体系全体を俯瞰的に理解したい場合は、ホワイトペーパーを参照されたい。

ホワイトペーパーでは、本方法論の背景、工程上の位置づけ、適用範囲、公開範囲を整理する。

### Technical Notes｜技術メモ

継続的に更新される工程観察、システム設計上の考察、応用拡張に関する内容を確認したい場合は、技術メモを参照されたい。
